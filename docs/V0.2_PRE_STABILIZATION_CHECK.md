# v0.2-pre Stabilization Check

## 中文

Updated: 2026-06-02

### 范围

本轮检查聚焦 v0.2-pre standalone runtime / packaged app / local data / bundled knowledge / reset controls / CI readiness。没有新增产品功能，没有改 persona prompt、provider 业务逻辑、model routing、semantic extraction、game session 语义、memory 语义、knowledge matching 或 proactive 规则。

### 自动化验证结果

- `make doctor`：通过。初始环境提示 8000 端口已有手动 backend，占用已在 packaged smoke test 前清理。
- `make validate-knowledge`：通过。已检查 6 个游戏，2 个 supported knowledge packs；planned 游戏缺知识包仅 warning。
- `make lint`：通过。
- `make test-backend`：通过，275 tests passed。
- `make test-desktop`：通过，renderer 44 tests passed，main 21 tests passed。
- `make typecheck`：通过。
- `make package-backend`：通过，PyInstaller 生成 macOS arm64 backend binary。
- `make package-desktop`：通过，生成 `apps/desktop/release/ReiLink-darwin-arm64/ReiLink.app`，并打入 backend binary、bundled knowledge 和只读 runtime resources。
- `git diff --check`：通过。

本轮发现并修复两个小稳定性问题：

- Desktop main process tests 之前会真实探测本机 8000 端口，导致开发 backend 正在运行时单测误判 `port_occupied`。已改为单测默认 mock 端口未占用，并保留专门的 port occupied 覆盖。
- Packaged backend 在 `REILINK_DATA_DIR` 指向用户数据目录时会从用户数据目录读取 persona，导致聊天失败 `Persona not found: rei_like`。已新增 `REILINK_RESOURCE_DIR`，并将 persona / persona style / game registry 等只读 runtime resources 打入 `.app/Contents/Resources`。memory / session / settings / logs 仍写入用户数据目录。

### Packaged App Smoke Test

Case A：无外部 backend

- 前置：确认 8000 端口无监听。
- 打开：`open apps/desktop/release/ReiLink-darwin-arm64/ReiLink.app`。
- 结果：UI 非黑屏；主聊天区、输入框、Settings、右侧面板可见。
- 后端：自动启动 `.app/Contents/Resources/backend/reilink-backend`；`/api/health` 返回 `{"status":"ok"}`。
- Runtime UI：后端来源显示内置后端，知识资源显示内置知识资源。
- 退出：退出 App 后 8000 端口释放。

Case B：外部 backend 已运行

- 前置：手动运行 `make dev-backend`。
- 结果：packaged app 显示“已检测到外部后端 / 外部后端”。
- 进程：未启动第二个 `reilink-backend`；8000 只由手动 uvicorn backend 占用。
- 退出：退出 App 后手动 backend 仍在运行。
- 清理：停止手动 backend 后，8000 端口无残留。

### Local Data 检查结果

- Packaged app 自启动 backend 时，Settings 显示用户数据目录为 `~/Library/Application Support/ReiLink/data`。
- Local Data 区显示 memory / session / settings / logs 均位于用户数据目录下。
- 点击“打开本地数据目录”成功打开 Finder。
- `.app/Contents/Resources` 内未发现 `.env`、memory 或 session 产物。
- reset UI 显示“重置演示状态不会清空长期记忆。危险操作会先确认。”
- Desktop tests 覆盖了危险 reset confirmation，以及 one-click demo reset 不清空长期记忆。

注意：如果 packaged app 连接的是手动启动的外部 backend，Local Data API 会显示该外部 backend 自己的 data dir。当前 repo-local `make dev-backend` 默认使用 repo `data`，这是外部 backend 的预期行为。

### Knowledge Pack 检查结果

- Packaged app 自启动 backend 时，Local Data API 显示 `knowledge_source=bundled`、`using_bundled_knowledge=true`。
- 手动设置 `elden_ring` 后询问 `Margit 怎么打`：`knowledge_available=true`，命中 3 条 snippets，`knowledge_used_in_prompt=true`。
- 手动设置 `hollow_knight` 后询问 `螳螂领主怎么打`：`knowledge_available=true`，命中 1 条 snippet，`knowledge_used_in_prompt=true`。

### GitHub / Public Readiness

- `.github/workflows/test.yml` backend job 使用 `LLM_PROVIDER=mock` 和空 `DEEPSEEK_API_KEY`，不依赖真实 DeepSeek key。
- Desktop CI 运行 lint / test / build。
- E2E job 仍是 `workflow_dispatch` 手动触发。
- README 包含 screenshots、MIT License、免责声明和隐私 / 本地数据说明。
- `docs/RELEASE_NOTES_v0.2-pre.md`、`docs/PROJECT_STATUS.md`、`docs/roadmap.md` 已与 standalone runtime 行为同步。
- `docs/PUBLIC_READINESS_CHECKLIST.md` 已更新为当前 v0.2-pre 已发布状态。

### 当前限制

- backend binary 由 PyInstaller 生成，当前是 macOS 本地未签名开发构建。
- 当前没有 installer / DMG、code signing、notarization 或 auto updater。
- Windows / Linux packaging 尚未完成。
- API key 仍通过本地环境或 `.env` 配置，不会内置进 App。
- 外部 backend 模式下，数据目录由外部 backend 决定；packaged app 不会接管或杀掉外部 backend。
- GitHub Actions 最新远端运行结果未在本轮通过 GitHub API 检查。

### 后续建议

- 建议为本轮 runtime stabilization 创建新的 v0.2-pre runtime tag；不要重写已经公开的 `reilink-v0.2-pre` tag。
- 下一步优先做 macOS installer / DMG spike、code signing / notarization research，以及 Windows packaging 调研。
- 后续进入 Voice / RAG / Overlay / Live2D 前，先保持 standalone runtime 和 Local Data 行为稳定。

## English

Updated: 2026-06-02

### Scope

This pass focused on v0.2-pre standalone runtime, packaged app behavior, local data, bundled knowledge, reset controls, and CI readiness. It did not add product features and did not change persona prompt content, provider business logic, model routing, semantic extraction, game session semantics, memory semantics, knowledge matching, or proactive rules.

### Automated Verification Results

- `make doctor`: passed. The initial environment reported a manually started backend on port 8000; it was cleared before the packaged smoke test.
- `make validate-knowledge`: passed. 6 games checked, 2 supported knowledge packs; planned games without packs only produced warnings.
- `make lint`: passed.
- `make test-backend`: passed, 275 tests passed.
- `make test-desktop`: passed, renderer 44 tests passed and main 21 tests passed.
- `make typecheck`: passed.
- `make package-backend`: passed, producing a macOS arm64 PyInstaller backend binary.
- `make package-desktop`: passed, producing `apps/desktop/release/ReiLink-darwin-arm64/ReiLink.app` with the backend binary, bundled knowledge, and read-only runtime resources.
- `git diff --check`: passed.

Two small stabilization issues were found and fixed:

- Desktop main process tests previously probed the real local port 8000, so they could misreport `port_occupied` when a development backend was already running. Tests now mock the closed-port path by default while retaining a dedicated port occupied case.
- The packaged backend read persona data from the writable `REILINK_DATA_DIR`, so packaged chat could fail with `Persona not found: rei_like`. A read-only `REILINK_RESOURCE_DIR` was added, and persona / persona style / game registry resources are bundled into `.app/Contents/Resources`. Memory, session, settings, and logs still go to the user data directory.

### Packaged App Smoke Test

Case A: no external backend

- Precondition: port 8000 had no listener.
- Launch: `open apps/desktop/release/ReiLink-darwin-arm64/ReiLink.app`.
- Result: the UI was not black; main chat, input, Settings, and right-side panels rendered.
- Backend: the app auto-started `.app/Contents/Resources/backend/reilink-backend`; `/api/health` returned `{"status":"ok"}`.
- Runtime UI: backend source showed bundled backend, and knowledge source showed bundled knowledge.
- Quit: port 8000 was released after quitting the app.

Case B: external backend already running

- Precondition: `make dev-backend` was started manually.
- Result: the packaged app showed external backend detected.
- Process check: no second `reilink-backend` process was started; port 8000 belonged only to the manual uvicorn backend.
- Quit: quitting the app did not kill the manual backend.
- Cleanup: after stopping the manual backend, port 8000 had no listener.

### Local Data Results

- With the app-started packaged backend, Settings showed the user data directory as `~/Library/Application Support/ReiLink/data`.
- The Local Data section showed memory, session, settings, and logs under the user data directory.
- The "打开本地数据目录" button opened Finder successfully.
- `.app/Contents/Resources` did not contain `.env`, memory, or session output.
- The reset UI states that demo reset does not clear long-term memory and dangerous operations confirm first.
- Desktop tests cover dangerous reset confirmation and one-click demo reset preserving long-term memory.

Note: when the packaged app connects to a manually started external backend, the Local Data API shows that external backend's own data dir. The repo-local `make dev-backend` default is repo `data`, which is expected for external backend mode.

### Knowledge Pack Results

- With the app-started packaged backend, Local Data API reported `knowledge_source=bundled` and `using_bundled_knowledge=true`.
- After manually setting `elden_ring`, asking `Margit 怎么打` produced `knowledge_available=true`, 3 matched snippets, and `knowledge_used_in_prompt=true`.
- After manually setting `hollow_knight`, asking `螳螂领主怎么打` produced `knowledge_available=true`, 1 matched snippet, and `knowledge_used_in_prompt=true`.

### GitHub / Public Readiness

- `.github/workflows/test.yml` uses `LLM_PROVIDER=mock` and an empty `DEEPSEEK_API_KEY` for backend CI, so it does not require a real DeepSeek key.
- Desktop CI runs lint / test / build.
- The E2E job remains a manual `workflow_dispatch` job.
- README includes screenshots, MIT License, disclaimers, and privacy / local data notes.
- `docs/RELEASE_NOTES_v0.2-pre.md`, `docs/PROJECT_STATUS.md`, and `docs/roadmap.md` are aligned with standalone runtime behavior.
- `docs/PUBLIC_READINESS_CHECKLIST.md` has been updated for the current published v0.2-pre state.

### Current Limitations

- The backend binary is generated by PyInstaller and currently targets unsigned local macOS development builds.
- There is no installer / DMG, code signing, notarization, or auto updater yet.
- Windows / Linux packaging is not complete.
- API keys are still configured through the local environment or `.env`; they are not bundled into the app.
- In external backend mode, the external backend owns its data directory; the packaged app does not take it over or kill it on quit.
- The latest remote GitHub Actions run was not checked through the GitHub API in this pass.

### Recommendations

- Create a new v0.2-pre runtime stabilization tag for this pass; do not rewrite the already public `reilink-v0.2-pre` tag.
- Next focus should be macOS installer / DMG spike, code signing / notarization research, and Windows packaging research.
- Keep standalone runtime and Local Data behavior stable before entering Voice, RAG, Overlay, or Live2D work.
